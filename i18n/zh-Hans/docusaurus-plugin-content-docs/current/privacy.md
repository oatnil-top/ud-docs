# 隐私政策

*一个独立开发者关于数据隐私的真心话*

*最后更新：2025年12月*

## 说点真心话

我是一个独立开发者，不是什么大公司。我做 UnderControl 的一大初衷，就是想把自己的敏感数据——任务、财务、个人笔记——完全掌控在自己手里，而不是交给大公司。我先是为自己做的，现在分享给大家。

我对你的数据没有任何兴趣、能力、精力或动机去做什么。话虽如此，我理解你可能不完全信任我——你本来就不应该盲目信任网上的任何人。那些有专业安全团队的大公司都会被黑客攻破，你凭什么信任一个独立开发者？你完全有权保持怀疑。

## 你的选择

所以我给你选择。你不必使用云服务。你可以用**桌面应用**——数据100%留在你的设备上，永远不会碰到任何服务器。或者你可以**自托管**——运行自己的实例，完全掌控，你的基础设施。

云服务存在是为了方便，不是因为我想要你的数据。如果隐私是你的首要考虑，就用本地版或自托管。我完全理解。

## 服务器上保存什么

如果你使用云服务：账户信息（邮箱、用户名）和你创建的数据——任务、支出、预算等。我承诺不会查看你的数据，但你不必信任我。

> **选择桌面应用或者自部署，完全控制自己的数据**

## 安全性

我尽力做好：全站 HTTPS，密码加密存储，数据库有安全防护。但我不是安全专家。如果你处理敏感数据，建议自托管或使用桌面应用。

对于自托管用户，安全掌握在你自己手中。保持服务器更新，遵循基本的安全实践。

## 你的选项

| 选项 | 说明 |
|------|------|
| **桌面应用** | 100% 本地，仅存储在你的设备 |
| **自托管** | 你的服务器，完全掌控 |

## Web Clipper Chrome 扩展

[Web Clipper](/docs/web-clipper) Chrome 扩展有额外的隐私说明：

- **网站内容**仅在你明确点击"保存"时捕获。内容保存到本地磁盘或发送到你自己的 UnderControl 服务器。不会将任何内容发送给第三方。
- **身份验证凭据**存储在你的设备本地。仅传输到你配置的服务器地址。
- 扩展**不使用**远程代码。所有脚本均打包在本地。
- 扩展**不会**跟踪浏览历史或用户活动。
