# 版本发布记录

UnderControl 的版本更新历史和新功能介绍。

## 版本号说明

UnderControl 采用**语义化版本号**（格式：主版本号.次版本号.修订号），例如 `v0.19.0`：

- **主版本号 0**：表示开发版本，API 和功能可能会有较大变动
- **次版本号**（如 0.**19**.0）：增加新功能、重大改进或包含破坏性变更时递增
- **修订号**（如 0.19.**0**）：修复 bug、小优化等向后兼容的改进时递增

## v0.108.0 (2026-07-08)

### 安全

- 修复高级搜索中的一个 SQL 注入漏洞：精心构造的自定义字段名可能被注入到底层查询语句中。

### 新功能

- **统一搜索栏** —— 高级搜索现在整合为一条查询栏，含三种模式：**筛选**（可视化标签）、**查询**（类 SQL 语句）、**问 AI**（用自然语言描述）。筛选条件即增即用、即时生效；在「筛选」和「查询」之间切换不会丢失已有条件；还有实时有效性提示，告诉你查询是否可用。
- **看板列支持按更多字段筛选** —— 看板列现在不仅能按状态、标签，还能按**标题**、**描述**等字段匹配，并新增「同时包含多个标签」选项。列编辑器会实时预览生成的查询语句，并为模糊匹配给出通配符提示。

### 改进

- 看板：按只读字段（如标题）筛选某一列时，不会再在卡片拖入该列时误改这些字段。

### 问题修复

- 修复任务详情页终端（tty）的对齐和缩放问题。

## v0.107.0 (2026-07-06)

### 新功能

- **智能体团队** —— 把 AI 智能体编成团队（一个负责人 Lead + 若干专长成员）。每个成员都带一条「何时委派给它」的说明，这份名册会自动注入到 Lead 的会话里，让它知道该把哪类活儿交给谁；团队还能嵌套成委派层级。在全新的 **智能体** 中心（智能体 · 团队 · 技能 三个标签页）里管理，或用 `ud` CLI（`ud apply` / `ud get agent-team`）操作。
- **复制为富文本** —— 右键任务或其描述，即可复制为富文本，方便粘贴到其它应用。

### 改进

- 评论线程：解决线程时，操作从原来的「解决 + 删除」改为 **解决 + 标记完成 + 终止**。
- 文件浏览器：附加到任务上的资源，现在会和该任务并列展示。

- 数据流：修复重命名/复制/删除时的撤销与数据损坏问题、图标节点锚点跳动、以及任意方向连接点的管道渲染

```sql
-- 可安全重复运行（使用 INSERT OR IGNORE）

INSERT OR IGNORE INTO resource_entity_links (
    id,
    resource_id,
    entity_type,
    entity_id,
    created_at,
    created_by,
    updated_at,
    updated_by
)
SELECT
    lower(hex(randomblob(4)) || '-' || hex(randomblob(2)) || '-4' ||
          substr(hex(randomblob(2)),2) || '-' ||
          substr('89ab',abs(random()) % 4 + 1, 1) ||
          substr(hex(randomblob(2)),2) || '-' || hex(randomblob(6))) as id,
    r.id as resource_id,
    r.entity_type,
    r.entity_id,
    COALESCE(r.created_at, datetime('now')) as created_at,
    r.owner_id as created_by,
    COALESCE(r.updated_at, datetime('now')) as updated_at,
    r.owner_id as updated_by
FROM resources r
WHERE r.entity_type IS NOT NULL
  AND r.entity_type != ''
  AND r.entity_id IS NOT NULL
  AND r.entity_id != ''
  AND NOT EXISTS (
    SELECT 1 FROM resource_entity_links rel
    WHERE rel.resource_id = r.id
      AND rel.entity_type = r.entity_type
      AND rel.entity_id = r.entity_id
  );
```

**运行方法**：
1. 使用 SQLite 客户端连接到数据库文件
2. 执行上述 SQL 脚本
3. 重启应用
