我是建筑学出身，后来转行做了程序员，再后来离开熟悉的环境，出海新加坡——现在在这边做全栈 / DevOps 工程师。

### 转行的人，最懂"积累"两个字

转行意味着从零开始：没有科班四年打底，每一个知识点都要自己啃。那段时间我养成了一个习惯——把踩过的坑、看懂的概念、调通的配置，全都记下来。不是因为勤奋，是因为怕：今天不记，明天就得再花一遍时间。

现在回头看，这个习惯是我做过的回报率最高的投资。转行靠它，跳槽靠它，出海靠它，后来做自己的产品还是靠它。知识和钱一样，是有复利的——前提是你得有一个能让它"生息"的账户。

这些年里，我的"账户"换了三次：Joplin → Obsidian → UnDercontrol。每一次迁移，都不是喜新厌旧，而是我对"知识应该怎么工作"的理解升了一级。

![两段旅程，一个习惯](https://pub-35d77f83ee8a41798bb4b2e1831ac70a.r2.dev/features/blog/knowledge-compounding/concept-2.png)

### Joplin 时代：先把东西存下来（Collect）

我的第一站是 Joplin。开源、免费、支持 Markdown、能加密同步——对一个刚开始认真做笔记的人来说，它解决了最重要的问题：**让"记下来"这件事发生**。

转行初期的学习笔记、面试准备、工作中排查问题的记录，都是从这里开始堆起来的。

但用久了会发现，笔记像一个个抽屉：存进去容易，但每条笔记都是孤立的。笔记本身存在应用的数据库里，找东西靠文件夹和关键词搜索，笔记之间没有真正的关联。知识存下来了，但只是"躺着"——它不增值。

### Obsidian 时代：让知识连接起来（Connect）

于是我搬到了 Obsidian：纯本地 Markdown 文件、双向链接、知识图谱、丰富的插件生态。

这是一次真正的升级。笔记第一次从"一堆文件"变成了"一张网络"——写新东西的时候，旧笔记会自然地被链接进来。我的笔记数量在这个阶段破了千，复盘、写作、做技术决策，都能感觉到过去的积累在托着我。

转行和出海，这两次人生里最大的跳跃，背后都是同一批笔记在支撑。这是我第一次真切地感受到"知识的复利"不是一句鸡汤。

但积累越深，新的裂缝也越明显：

- **知识被锁在本地文件夹里。** 最典型的场景：在公司电脑前 debug 一个棘手问题，我突然想起这个坑我踩过，详细记录在家里那台电脑的 Obsidian 里——然后就没有然后了。我不可能把整个知识库同步到每台设备上。
- **图谱是个美丽的谎言。** 只要正文里提到 `[[某个概念]]`，它就进图谱。随手一提的临时引用和精心构建的知识关联，在图上是等权重的边。节点越多，图越好看，信息量却越低。
- **附件散落各处。** 架构图、流程图、PDF 散在各个 assets 文件夹里，文件夹一挪，链接就断。
- **同步方案各有各的痛。** 付费同步依然要装客户端；Syncthing 没法装在公司电脑上；Git 同步技术上可行，体验上是折磨。

我需要的其实很简单：**一个网页，登录即用，完整的编辑能力，知识随时可及。**

### UnDercontrol：让知识可以被调用（Compound）

市面上没有刚好长成我想要的样子的工具，所以我决定自己造一个。这也是一个实验：一个人加上 AI，能把一个产品做到什么程度？

UnDercontrol 是我对前两个阶段所有裂缝的回答：

- **随处可及**：浏览器登录即用；同时有 CLI 和桌面端，`ud pull / ud push` 可以把任务双向同步成本地 Markdown 文件——Web 的便利和本地文件的安心，两个都要。
- **链接由我显式构建**：只有主动建立的关联（peer / parent-child）才进知识图谱，每条边都有意义，信噪比高。
- **资源集中管理**：图片、PDF、附件跟着具体的上下文走，但可以在统一的资源库里浏览、检索、复用。
- **不只是任务**：UnDercontrol 的 Markdown 编辑器是所有文本表面共用的——任务、笔记、财务记录、文档，同一套编辑体验。一条任务可以是一篇文档、一次决策记录、一篇博客草稿。
- **AI 原生**：知识库对 AI 是开放的。Claude Code、Codex、OpenCode 或任何终端 Agent，都能通过 CLI 读写同一份知识。事实上，你现在读的这篇博客，就是一个 AI Agent 在 UnDercontrol 的任务里，读着我过去的笔记写出来的。

这是复利的第三个阶段：知识不再只是"存着"或"连着"，而是能**被调用**——被未来的我调用，也被替我干活的 AI 调用。

### 时间沉淀出力量

从建筑图纸到代码，从国内到新加坡，从 Joplin 到 Obsidian 再到自己写一个工具——变的是职业、城市和软件，不变的是那个习惯：**把今天想明白的东西写下来**。

复利的本质不是收益率，是不中断。工具的意义，就是让"不中断"这件事变得足够容易。

UnDercontrol 对我来说，既是产品，也是这个习惯的容器。我每天用它管理自己的任务、笔记和财务，也用它管理"做 UnDercontrol"这件事本身。我和它一起成长，它替我记住时间沉淀下来的一切。

如果你也在 Joplin、Obsidian 和无数工具之间迁移过，如果你也相信知识有复利——欢迎来试试 [UnDercontrol](https://oatnil.com)。
